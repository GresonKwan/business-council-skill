# Mind Card Template / 思维卡片模板

本文件是 `business-council` skill 中所有思维卡片的标准格式。新增人物时，请按此模板填充，确保卡片具备**可复现性、可区分性、可修正性、可扩展性**。

---

```markdown
---
card-meta:
  name: [人物全名]
  identity: [主要身份 / 标签]
  active-era: [活跃阶段 / 年代]
  scope: [聚焦领域：战略 / 产品 / 资本 / 组织 / 危机]
  domain: [适用行业或情境]
  version: "1.0"
  last-reviewed: "YYYY-MM-DD"
  sources:
    - [来源1 标题](URL)
    - [来源2 标题]
---

## [人物全名] — [阶段/情境标签]

### 核心权重 / Core Weights

用 `X > Y` 表达决策优先级。每条注明适用情境。

1. **X > Y**：...[适用情境：...]
2. **X > Y**：...[适用情境：...]
3. **X > Y**：...[适用情境：...]

### 心智模型 / Mental Models

每个模型必须包含：签名、定义、触发条件、执行步骤、边界/反例、高成本坚持点、案例锚点。

#### 1. [模型名称 / Model Name]
- **模型签名 / Signature**: `[唯一标识，如 Focus=SayingNo]`
- **定义 / Definition**: ...
- **触发条件 / Trigger**: 在什么情境下调用这个模型？
- **执行步骤 / Steps**:
  1. ...
  2. ...
  3. ...
- **边界与反例 / Anti-case**:
  - 不要在 X 情境下使用，因为...
  - 更安全的替代框架：...
- **高成本坚持点 / Persistence Point**: 他曾为坚持这个模型付出过什么代价？
- **案例锚点 / Anchor**: ...

#### 2. [模型名称 / Model Name]
...

### 风险与失败观 / Risk & Failure

- 他如何定义风险？
- 他容忍什么样的失败？
- 他从失败中如何学习？

### 人才与组织观 / People & Organization

- 他如何选人、用人、育人？
- 他喜欢什么样的组织形态？
- 他如何处理内部冲突？

### 竞争与产品观 / Competition & Product

- 他如何看待竞争？
- 他如何做产品决策？
- 他如何理解客户需求？

### 资本与时间观 / Capital & Time

- 他如何分配资本？
- 他的时间尺度是什么？
- 他如何权衡短期与长期？

### 语言与修辞风格 / Voice & Rhetoric

- 他的表达习惯是什么？
- 他如何说服别人？
- 中文表达时应注意什么？

### 典型问题 / Typical Questions

- 他会问什么问题来切入一个问题？
- 这些问题反映了他的哪些权重？

### 经典案例锚点 / Case Anchors

- [案例1]：...[说明其反映的权重或模型]
- [案例2]：...

### 高成本坚持点 / High-Cost Persistence Points

1. **[坚持点名称]**：[具体行为] — [付出的代价/成本] — [揭示的权重]
2. **[坚持点名称]**：...

### 可靠来源 / Sources

- [来源1 标题](URL) — [类型：股东信/访谈/演讲/传记/决策案例]
- [来源2 标题](URL) — [类型]

### 使用禁忌 / Taboos

- 不要把他/她简单化成一个标签。
- 不要在他/她不擅长的领域强行套用。
- 不要用单一事件概括其一生思维。
- [具体禁忌1]：...
- [具体禁忌2]：...

### 时间切片索引 / Time-Slice Index

- `[标签1]` → 见 `TIME_SLICES.md#[锚点]`
- `[标签2]` → 见 `TIME_SLICES.md#[锚点]`
```

## 填写质量自检

- [ ] 元信息块完整（身份、阶段、领域、版本、来源）
- [ ] 每条核心权重都注明了适用情境
- [ ] 每个心智模型都有触发条件、执行步骤、边界/反例
- [ ] 至少记录了一个高成本坚持点
- [ ] 来源可验证，并区分了事实与推断
- [ ] 使用禁忌具体，而不是泛泛而谈
- [ ] 已链接到对应的时间切片变体
