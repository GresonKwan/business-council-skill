# Decision Trace Template / 决策追溯模板

本文件用于记录历史回测（backtests）和用户自己的决策日志。每一个决策追溯都应包含：背景、问题、各视角预测、实际选择、结果、可修正性记录。

---

```markdown
### [ID]: [决策名称 / 事件]（[年份]）

- **决策者 / Thinker(s) invoked**: [@Jobs] / [@Musk] / [@Bezos] / [@Huang] / [Council]
- **决策背景 / Context**: 
  - 公司/人物当时处于什么阶段？
  - 外部约束是什么？（资金、竞争、监管、技术成熟度）
  - 内部约束是什么？（组织规模、文化、现金流）
- **当时的问题 / Question at the time**: 用一句话还原当时需要回答的核心问题。
- **历史实际选择 / Actual historical decision**: 当时实际做了什么决定？

#### 参谋团预测 / Council Predictions

（基于该人物当时的思维卡片或时间切片变体，模拟他可能会给出的判断）

- **Steve Jobs 视角**: ...
- **Elon Musk 视角**: ...
- **Jeff Bezos 视角**: ...
- **Jensen Huang 视角**: ...

#### 结果 / Outcome

- **短期结果**（0-12 个月）：...
- **中期结果**（1-3 年）：...
- **长期结果**（3 年以上）：...

#### 可修正性记录 / Correctability Notes

- 哪些预测与历史结果一致？反映了哪些权重或模型？
- 哪些预测与历史结果不一致？是卡片错误、情境差异，还是信息不足？
- 这次回测后，应该更新哪张卡片的哪个部分？

#### 来源 / Sources

- [来源1 标题](URL) — [类型]
- [来源2 标题](URL) — [类型]
```

## 使用说明

### 历史回测 / Backtests

用于验证思维卡片的准确性。选择公开、可验证的重大商业决策，还原当时的约束条件，看 skill 输出是否与历史选择一致。

### 用户决策日志 / User Decision Log

用户可以把自己的重大决策按此模板记录，日后复盘：

```text
/business-council
请帮我为以下决策建立一个决策追溯条目：
[决策背景]
[问题]
[实际选择]
[当前已知结果]
```

Agent 会按模板输出并建议是否更新相关思维卡片。

---

## 记录原则

1. **情境优先**：必须还原当时的约束，不能用后见之明替代当时的判断。
2. **来源可考**：每个关键事实都要能追溯到公开资料。
3. **诚实标记**：明确区分“事实”、“合理推断”和“反向推演”。
4. **行动导向**：每次回测都应产生至少一条对思维卡片的修正建议。
